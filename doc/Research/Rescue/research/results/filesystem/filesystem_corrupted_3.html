Bart Dorsey wrote (and I give him all the credit):

Sounds like bad sectors. :(
You can boot using the redhat boot.img and rescue.img disks...
then you will be able to run e2fsck

Another way to do it is to type
linux init=/bin/sh
which will boot up with only a shell running and the harddrive still
read only...
at that point you can run e2fsck -c /dev/whatever to check the drive.

The -c makes it run the "badblocks" program. which will mark those
blocks as being bad.

>
