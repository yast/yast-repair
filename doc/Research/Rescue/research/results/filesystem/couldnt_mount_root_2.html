I got a similar message as I was trying to build a new RH6.1 kernal on
my box.  In my case the original kernal build in the install process was
set to the defaults (appropriate for a newbie).  In the course of using
make xconfig I altered some disk parameters that caused one of two
events:

endless loop on a device probe (couldn't stop the screen updates to see
the entire message) 

or

kernal panic VFS ...

My solution was to replace the .config file in the usr/src/linux(forgot
the version number) directory with a 386 configuration file in one of
the source directories.  I changed the name which started with the
processor family and ended with .config to just .config in the correct
directory, rebuilt kernal and modules per the redhat reference help,
added it to LILO and the configuration worked.

Your problem sounds similar.

I presume that during your initial installation you made some changes to
the kernal, built it, and now nothing works.  

If you did not make any changes to your kernal during installation, it
is possible the default kernal has a problem with your disk controller.

Based on my limited experience, I would probably reinstall off the
CDROM, paying attention during the kernal building as to the selection
of hardware needed.  More experienced users may have better suggestions.

BTW, I run Win95 on my K2-400 computer.  RH6.1 is on the second
partition on the second HD.  No problems with that for me.

Hope this helps.

Pete Attas
