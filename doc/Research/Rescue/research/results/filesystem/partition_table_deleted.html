I don't see how "fsck" will help you.  "fsck" repairs file systems
by checking the super-blocks and i-node blocks for consistancy.
The partition table itself is used to define the bounds of the 
file-systems.  From you description, it is your partition table 
that is hosed.  The files system is gone but that is only because
the partition table is gone.  

Either Mike or I don't understand your situation.  
I rather think it's Mike.

My approach to this would be:

1.  Boot win95 

2.  Use Win95 fdisk to re-write the partition table such that
    you have 1 large partition over the area where Linux was.
    (I am assuming that Win95 is on the lower partition(s).)

3.  Arrange to have this disk accessable to a system which
    can boot linux or any other form of Unix. 

4.  Without doing a "mkfs" or any writes to the partition,
    use the Unix tool "dd" to read the partiton as an unmounted
    character device.  Send "dd" output to a file.  This means
    you will need a large amount of free disk space on the 
    recovery machine.

The data is still there, your system just does not know how to find
it.  If you wrote down all the start and end points that were
displayed in Linux fdisk earlier, then it would be easy to set the
same partitions back like they were.  As long as you did not write any
or format anything in that area of the disk, it would magically
reappear.

But if you do not know where or how big your partitions were, you may
be out of luck.
