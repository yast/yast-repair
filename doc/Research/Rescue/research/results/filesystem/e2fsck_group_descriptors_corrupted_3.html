Subject: Hard drive extremely unclean. Help!
Newsgroups: comp.os.linux.help, alt.os.linux, comp.os.linux.misc

Date: 1996/01/27 


Hi! My linux hard-drive (a 1 Gig IDE) seems really messed up. I'm wondering
if it's salvageable or not; here's the symptoms:

When I try this:
	mount -o ro /dev/hda2 /poo

I get:
EXT2-fs error (device 3/2): ext2_check_descriptors: Inode table for group 105
not in group (block 870844)!
EXT2-fs: group descriptors corrupted !
mount: wrong fs type or bad superblock on /dev/hda2

My kernel is version 1.3.44.

So I tried this:
	e2fsck /dev/hda2

and got:
e2fsck 0.5b, 14-Feb-95 for EXT2 FS 0.5a, 95/03/19
Warning: Inode table 870844 for group 105 not in group.
WARNING: SEVERE DATA LOSS POSSIBLE.
Continue<y>? yes

Warning: Inode table 951739 for group 115 not in group.
WARNING: SEVERE DATA LOSS POSSIBLE.
Continue<y>? yes

/dev/hda2 contains a file system with errors, check forced.
Pass 1: Checking inodes, blocks, and sizes
Relocating group 105's inode table to 870844
Relocating group 115's inode table to 951739
Restarting e2fsck from the beginning...
Warning: Inode table 870844 for group 105 not in group.
(... and so it continues, constantly relocating to the same place and
restarting.)

I gave up after e2fsck looped about 4 times.

At one point, the superblock was corrupted, but when I re-ran e2fsck with
-b 8193, I got the same errors as above.

Before all this (while my drive still seemed to be working ok), I had gotten
some bad blocks, as follows:
EXT2-fs panic (device 03:02): ext2_write_inode:
unable to read i-node block - inode = 214402, block = 860193

EXT2-fs panic (device 03:02): ext2_write_inode:
unable to read i-node block - inode = 234722, block = 946702

After the panic I had rebooted, and (perhaps foolishly?) tried to map out
those blocks: I booted with a Kernel 1.3.18 disk and a "rescue-disk" boot
floppy, created a file ("baddies") listing the two bad blocks, and ran:
	e2fsck -l /baddies /dev/hda2

... during which it asked if I wanted to relocate some data (I said yes),
and if I wanted to ignore some "can't read block (or inode)" errors (I said
no.) When I said "no" to ignore one of those errors, e2fsck quit. So I ran
it again, saying "yes" to "ignore error" questions.

Then I eventually got a bad superblock, and arrived at the point illustrated
at the top of this message.

Any pointers?

- Andy
