Subject: Adaptec 1542B and Disks >1GB
Newsgroups: comp.os.linux.hardware
(This is the only article in this thread)
Date: 1995/07/20 



I have Slackware 2.3 (July 1995) from Walnut Creek CD-ROM and I'm
trying to get an HP 1.2GB drive to work with my Adaptec 1542B
controller.

According to the Adaptec manual, non-DOS operating systems don't
require the 'Extended Translation' feature to be turned on. (Extended
translation is needed with DOS when drives larger than 1GB are used
since DOS has a 1024 cylinder limit. Extended translation translates
the drive's true geometry into one that appears to have 255 heads and
63 sectors and fewer than 1024 cylinders.) I have the new V3.2 Adaptec
firmware, but I have Extended Translation disabled.

I boot the Linux root disk and run fdisk on /dev/sda and everything
looks fine at first. Fdisk says the drive has 1149 cylinders. First I
created a Linux ex2 partition as partition 1 extendeding from cylinder
1 to cylinder 1133. Then I tried to create a swap partition on
cylinders 1134-1149, but fdisk warns that the swap partition doesn't
start on a cylinder boundary. The p command yields the following
display (paraphrased since I don't have the Linux machine here in
front on me).

Device	     Partition  Start  Begin  End    Type
/dev/sda1        1        1      1    1133    83  Linux Normal
/dev/sda2        2        1023   1134 1149    82  Linux Swap
warning: partition 2 doesn't start on a cylinder boundary

No matter what cylinder I tell fdisk to start partition 2 at, if it is
> 1023, fdisk says the partition 'starts' at cylinder 1023. This
doesn't seem correct. What is the difference between 'start' and
'begin' in fdisk's display?

I've read the SCSI HOW-TO and it doesn't mention anything about using
the 1542B with disks larger than 1GB. I was led to believe, however,
that the Linix 1542 driver automatically handles drives larger than
1GB. Based on the fdisk display and the warning, this doesn't seem to
be the case.

I'm also under the impression that the fiction of
cylinders/heads/sectors is used only by the Adaptec BIOS during the
boot process (or by DOS) and the Linux 1542 SCSI driver uses linear
block numbers. If this is the case, then why does fdisk seem to imply
a 1023 cylinder limit. In fact, when partitioning a SCSI drive, why
does it slice the drive up into 'cylinder' chunks anyway? Why not use
starting and ending block numbers?

Anyone know the answers? I'd like to use the upper 200MB on my drive...
