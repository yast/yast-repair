Hi,
  I'm trying to use a 360k 5.25" floppy drive on my PC (yes there is a
good reason to do this) which is running Linux 2.0.29. I installed it,
hooked it up and it runs OK under DOS (well I can do a directory of the
drive anyway). When I run linux and try to access it though, I get the
following messages:

> mdir a:
Can't open /dev/fd0: No such device or address
Cannot initialize 'A:'

I did have a disk in there, but it still wasn't working. I read some of
the Documentation files and a system manual I have, and tried the
following command:

# setfdprm -p /dev/fd0 360/360
# mdir a:
read: I/O error
init A: could not read boot sector
Cannot initialize 'A:'

I'm guessing that perhaps 360K drives aren't supported by linux? Do I
need to make a new entry in the /etc/fdprm file? (I thought the 360/360
would work, but the "size" is listed as 720 so this may not be right?)
Actually it gives the same error now as when there is no disk in the
drive. What am I doing wrong here and how can I fix it?

As a corollary, is there any way to run 3 floppy drives in a PC? (I want
to run a 360k 5.25", 1.2M 5.25" and 1.44M 3.5" simultaneously.

Thanks,

Duncan.
