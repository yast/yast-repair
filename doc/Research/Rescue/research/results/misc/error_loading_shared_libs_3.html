Subject: error loading shared libraries after upgrading to XF86 3.3.2 +3
Newsgroups: comp.os.linux.x
(This is the only article in this thread)
Date: 1998/10/13 


I was having trouble running XFree86 3.3.1 w/ RedHat 5.0 on my K6-2
300Mhz, Trident 3Dimage975 (AGP), and noticed that the gist of responses
to other users' problems w/ Trident cards was to upgrade to XFree86
3.3.2 patch 3, which says it supports many more Trident cards (the
3Dimage975 PCI is mentioned, but not AGP).
I downloaded x3323upd.tgz, x3323svga.tgz, and x3323vg16.tgz, along w/
extract, the XF86 extraction util.  I placed the four file in the
/var/tmp directory, as instructed by the Release notes for 3.3.2 p3;
then changed directories to /usr/X11R6 and ran the two commands:
/var/tmp/extract /var/tmp/x3323upd.tgz
/var/tmp/extract /var/tmp/x3323svga.tgz
(and then, actually repeated w/ vga16 server file):
/var/tmp/extract /var/tmp/x3323vg16.tgz

After doing this, when I try to run startx, or open xinit w/ emacs, or
view many files associated w/ starting or configuring X, I get the
error:
xinit:  error in loading shared libraries
/usr/X11R6/lib/libXext.so.6:  undefined symbol: _Xglobal_lock

anyone know what this means?  I found another thread (I think it may
have even been in a FAQ somewhere) which referred to a similar problem,
where both 6.1 and 6.0 versions of library files were existent, and the
fix was to copy the libXext.so.6.0 to libXext.so.6.0.sav and then make a
symbolic link from the 6.1 file to the 6.0 filename.  I do not have any
6.1 version of the libXext file, so I'm at a total loss for what to do.

Any help would be greatly appreciated.
