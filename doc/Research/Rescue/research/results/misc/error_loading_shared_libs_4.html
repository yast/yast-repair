Subject: XFree86 3.3.3 and error in loading shared libraries
Newsgroups: comp, comp.os.linux.x, comp.windows.x.i386unix
Date: 1998/12/12 


I am running Red Hat Linux 5.1 with kernel 2.0.35.
The graphics card is a 4MB S3Virge, using the XFree86_SVGA server.

I'm trying to upgrade to XFree86 3.3.3 which I downloaded from
www.xfree86.org.  I used the glibc version (not the libc5 version) since
Red Hat 5.1 is glibc based.  After installing the files I tried to
'startx' but I get the following error:

xterm: error in loading shared libraries
/usr/i486-linux-libc5/lib/libXt.so.6:  undefined symbol:  _Xsetlocale

I tried editing /etc/termcap (as the setup notes suggested) by replacing
the xterm entries with those in /usr/X11R6/lib/X11/etc/xterm.termcap,
but still I get the same error.

I also tried using the XFree86_S3V server, but still get the same error.

Have any of you experienced this error, and does anyone know of a fix?

Any help would be greatly appreciated.

WOW. I'm surprised so many people have had this same problem, and Redhat
has apparently not made any mention of it on their pages... well,  what I
did to "fix" it (and it's not really fixed completely for me yet....) is to
open up your
/etc/ld.so.conf file and make sure that /usr/i486-linux-libc5/lib is placed
AFTER the /usr/X11R6/lib entry. This should fix your problem with programs
using glibc, but as of now, I'm not having any luck with programs
requesting
libc5.

Hope this helps.

I've found a solution ( it worked for me ). I verified that the
version of the ldso file was 1.9.9 - 1. So I downloaded a newer
version: ldso 1.9.9 - 5.
After installing it I started Xfree86 3.3.3 with 'startx', and it went
just fine without giving any error message.
Try it, maybe it works also for you.
