Boot

The articles in this section refer to the boot process of a Linux system. These articles describe the handling of the different bootloaders and the creation of boot diskettes. The use and build of rescue diskettes is the topic of the articles in the rescue diskettes-section.

LILO

LILO (LInux LOader) is the default boot manager for Linux. Errors on LILO or it's configuration in most cases cause in an unbootable system (blocker errors).
  1. BOOT.LILO.1
    Error message: "ran out of input data":
    After starting LILO the boot process stops with the error message "ran out of input data". The system can only be started with a bootable diskette or a CD-ROM. LILO has to be reconfigured and restarted or a new kernel must be installed.
    References:
    SuSE Support Database or local,
    The Linux HOWTOs or local.

  2. BOOT.LILO.2
    System reboots:
    After starting LILO the system reboots automatically.
    References:
    SuSE Support Database or local.

  3. BOOT.LILO.3
    System freezes:
