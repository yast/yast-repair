I have a need to boot linux from a zip disk. The zip disk is not large
enough
to hold all of linux, so averything is on the zip disk except /usr.
There is a
symbolic link on the zip disk to link to /usr on the internal hard disk.
This all
works with RedHat 6.0. However. there are a couple of problems with xfs.

First, although on startup the init script claims to have started xfs, a
glance at
the processes shows it isn't running. As a consequence when I type
"startx"
it fails with a message that it cannot find the path to the default font
'fixed'.

So, my first question is: why does the init claim xfs has started but it
isn't running?

I can start xfs manually (actually I put this in startx) and after it is
running I can
type "startx" and everything is working.

I want to switch to RedHat 6.2. I reproduced everything under 6.2 that I
had done
under 6.0. The same problem exists with xfs not starting, but this time
even if I
start it up manually I still get an error when I type "startx" - the
error that there
is no path to the default font 'fixed'. The message is something like
this:

       _FontTransSocketUNIXConnect: Can't connect: errno = 111
     failed to set default font path 'unix/:-1'
     Fatal server error:
     could not open default font 'fixed'

When I boot from the hard disk everything works just fine - xfs gets
started by init
and startx launches properly.

So, my second question is: can anyone suggest a solution to this
problem? (I'd
also appreciate an explanation of the cause, but I'd settle for a
solution)

Thanks
