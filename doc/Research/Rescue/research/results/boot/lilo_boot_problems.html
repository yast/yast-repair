Boot problems with LILO (Kernel 2.0 or later)

Deutsch

Support knowledgebase (kfr_50)
Applies to

Kernel: Versions since 2.0

Problem

When booting with LILO problems appear after the kernel was loaded :-(

It is possible to choose a kernel and this kernel will be loaded then (some dots appear). However starting this kernel doesn't work. This means : The system crashes with a couple of different reactions before the message uncompresing Linux appears on the screen.

Possible error messages and phenomena :

  • System reboots
  • System simply freezes
  • crc-error
  • no free space
  • Error 0x00
  • Error 0x01
  • incomplete literal tree
Afterwards the floppy might be accessed from time to time, however the system is halted.

Cause 

The cause for this is a combination of huge kernels, LILO and faulty hardware. Approximately 1 % of the machines is affected.

We suspect that a faulty BIOS has problems with fast memory access.

The problem does not appear, if

  • the computer is booted with Loadlin,

  • the kernel is e.g. copied with dd if=/vmlinuz of=/dev/fd0 to a floppy and is directly loaded from there,

  • a smaller kernel, which was built with make zImage (e.g. an older 1.2.13 kernel) is booted with LILO.
Machines with the following BIOS settings could also be booted without problems:
  • Disabled Internal Cache
  • DRAM Precharge Wait State set to 1 and
    DRAM Wait Burst Timing to 0x3333

Solution

First it must be possible to install a system. If you cannot boot directly with Loadlin (or Setup.exe), you should initially an old 1.2.13 bootdisk use for the installation. If you do not have one available, rearrange the BIOS parameters.

After a successful installation the question arises, how one can boot the system. First one boots with the same medium as for the installation. There is no problems with using loadlin from a DOS partition. If you use a bootdisk then use the following parameters : 

        load_ramdisk=0 root=/dev/???
where ??? is your root partition.

Afterwards you should immediately generate a new kernel since this can then be booted with LILO.

If you have further questions about this, please send us the complete specification of your mainboard and your BIOS enclosed with your mail.
