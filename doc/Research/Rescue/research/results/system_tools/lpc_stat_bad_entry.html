"lpc stat" reports "Bad entry"

Deutsch

Support knowledgebase (stw_leerzeilen)
Applies to

SuSE Linux: Version 6.0

Symptom:

The printer does not print, when you type the command "lpc stat" you recieve the message: "Bad entry".

Reason:

When using YaST to set up a network printer, empty lines are inserted in between the entries in the /etc/printcap config file.

Solution:

Delete all empty lines in the file /etc/printcap, or at a minimum place a "#" infront of the empty lines. At the end of the file you must have one blank line, but only one!
