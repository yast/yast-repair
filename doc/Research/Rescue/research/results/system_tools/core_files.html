this is probably because xfm is trying to do some operation on /proc
which isn't allowed, and it can't recover properly. /proc is a special
filesystem which contains information about your system. this info can
be inspected with the command "procinfo" and others. you've no need to ever
go into this directory with the file manager.

"core" files are in general, useless. feel completely safe to delete them.
you can delete all "core" files on your system with (may take a while) :

find / -name "core" -exec rm {} \;

they are essentially a system dump and only ever useful in extreme situations.

you can set the max. size of a coredump by entering the following command in
/etc/profile or the like :

ulimit -Sc 0

this may not have any effect, however.
