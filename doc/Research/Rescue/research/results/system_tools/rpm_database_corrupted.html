RPM database corrupted

Deutsch

Support knowledgebase (ke_rpm-db)
Applies to

SuSE Linux: Versions since 5.0

Symptom:

It is not possible anymore to install software packages. A system update cannot be started or finished. A message like the following might appear : 

free list corrupt(1753864) - contact support@redhat.com.

Cause:

The RPM database in the directory /var/lib/rpm might be damaged.

Solution:

rpm can rebuild that database.

  1. Please make a backup of the directory /var/lib/rpm first.

  2. Execute the command 
    rpm --rebuilddb

  3. Try to continue the update procedure or install packages with YaST, if desired.
