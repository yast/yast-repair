PCI: System freezes after the kernel build while booting

Deutsch

Support knowledgebase (maddin_pci_kernel)
Applicable to
S.u.S.E. version: > 4.3
Kernel version: all

Symptom:

You have a computer system with PCI bus and have compiled an own kernel for your system.

Some devices in your system, like e.g. your Adaptec 2940 controller, are not detected and the root-file system cannot be mounted.

Cause:

Have you maybe forgotten during the kernel build the support for PCI systems?

Solution:

Start the system with the bootdisk enclosed with the product, like for the installation. Choose however "Start the installed system" and declare your / partition's device name.

Please log yourself in as root and compile a new kernel, that contains support for the installed devices. And of course support for PCI systems!

Please do not forget, to reinstall afterwards LILO.
