Subject: Problem setting up Ethernet card under Red Hat 7.0
Newsgroups: comp.os.linux.networking
Date: 2001-02-18 20:15:47 PST 

Hi,

I'm running a dual-boot system, booting Win 98 and Red Hat Linux 7.0
with a stock kernel.  I have a cable modem (AT&T RoadRunner), and I'm
trying to set that up under Linux.  It works fine under Win 98.  In
Linux, I used linuxconf to set up my Ethernet card to use DHCP and be
activated at bootup.  The card is a SMC Ethercard Elite16 Ultra card
(ISA), and I'm using the smc-ultra module as the driver.

However, I can't seem to get my Ethernet card to be activated under
Linux.  During startup, Red Hat reports the following error when it runs
the network script in /etc/rc.d/init.d/:

"Determining IP information for eth0...Operation failed."

At first, I thought it was a DHCP problem, but when I tried to run
'ifconfig eth0 up' I got the following error message:

"SIOCSIFFLAGS: Resource temporarily unavailable"

I'm not quite sure what this error means.  A Google search on this error
message found some suggestions about disabling PnP in the BIOS and
adding configuration information in /etc/modules.conf to make sure the
card gets the right IP address.  However, I tried disabling PnP in the
BIOS (not sure offhand what kind of BIOS I have, but my system is a
2-year old Gateway PII 450), and that didn't do anything.  And I already
have configuration info in modules.conf.  Here's the relevant info from
that file:

alias eth0 smc-ultra
options smc-ultra io=0x280 irq=11

The io and the irq settings are copied from my Win98 setup, where they
seem to work fine.

In case it's helpful, here's the output from 'ifconfig eth0':

eth0      Link encap:Ethernet  HWaddr 00:00:C0:E6:A0:D5  
          BROADCAST NOTRAILERS RUNNING  MTU:1500  Metric:1
          RX packets:0 errors:0 dropped:0 overruns:0 frame:0
          TX packets:0 errors:0 dropped:0 overruns:0 carrier:0
          collisions:0 txqueuelen:100 
          Interrupt:11 Base address:0x290 Memory:d0000-d4000

And my ifcfg-eth0 script:

DEVICE=eth0
IPADDR=
NETMASK=
ONBOOT=yes
BOOTPROTO=dhcp
IPXNETNUM_802_2=""
IPXPRIMARY_802_2="no"
IPXACTIVE_802_2="no"
IPXNETNUM_802_3=""
IPXPRIMARY_802_3="no"
IPXACTIVE_802_3="no"
IPXNETNUM_ETHERII=""
IPXPRIMARY_ETHERII="no"
IPXACTIVE_ETHERII="no"
IPXNETNUM_SNAP=""
IPXPRIMARY_SNAP="no"
IPXACTIVE_SNAP="no"
BROADCAST=
NETWORK=
USERCTL=no

Does anyone have any idea what the problem is?  I apologize in advance
if this is a RTFM question, but in that case, a pointer to the FM would
be appreciated.  I'm still a newbie when it comes to sysadmin stuff
under Linux.

Thanks in advance,

Jim Zawisza

The bios that is referred to is the NIC bios not the MB bios.  You have to
get a program from the NIC vendor to disable PnP on the card and set the io,
irq manually.  If the card is PnP now then it may work ok but Linux may get
a different irq and io than windows.  Try removing the irq and io from the
ifcfg-eth* file and let Linux autodetect.  If it still will not load then
try the manual route. If you change the NIC bios you will have to redo the
Win config as well.

Hope this helps

PatG

Never mind: I managed to get it working.  For the benefit of the
archives, the problem turned out to be an IRQ conflict between my sound
driver and the Ethernet driver.
