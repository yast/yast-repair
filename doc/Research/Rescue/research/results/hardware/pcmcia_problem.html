Subject: PCMCIA problem
Newsgroups: comp.os.linux.hardware, comp.os.linux.misc, comp.os.linux.setup
Date: 1997/11/12 



I have a no-name MMX laptop that locks-up when it hits the PCMCIA
section when booting. The boot display is as follows:

Starting PCMCIA services: modulesLinux PCMCIA Card Services 2.9.6
Intel PCMCIA probe:
  TI 1131 CardBus at 0x3e0, 2 sockets
    host opts:  [ring] [isa irq]
    irq mask (

At this point the machine is totaly unresponsive and must be rebooted
with the power switch.

I am running S.U.S.E 5.0. The kernel is 2.0.31. The
/etc/sysconfig/pcmcia file is as follows:

PCMCIAyes
PCICi82365
PCIC_OPTS
CORE_OPTS

The, heavly edited, /etc/pcmcia/config.opts file (without the comments)
is as follows:

include port 0x3e0-0x3e1, memory 0xcb000-0xccfff
exclude irq 4
exclude irq 3
exclude irq 5
exclude irq 7

The information from the, vary thin, manual gives the following
information:

irq 10 PCMCIA CardBus Controler
irq 11 PCMCIA CardBus Controler
I/O Port 03E0h - 03E1h PCMCIA controler
Memory Address 000CB000h - 000CBFFh PCMCIA controler
Memory Address 000CC000h - 000CCFFh PCMCIA controler

Anyone have an idea what's wrong?
-- 
Dave Blondell   n9jdz@xsite.net
Running Linux 2.0.31

Subject: PCMCIA problem
Newsgroups: comp.os.linux.misc
Date: 2000/06/27 


Can anyone help me sort out my PCMCIA problem?

I am running Linux Mandrake 6.5 (Kernel 2.2.13 I think) on a Dell Inspiron
3000 laptop but my PCMCIA cards (a Psion Gold v90 56k modem and a Xircom
10/100 Ethernet) are not available.

I have reinstalled the kernel-pcmcia-cs RPM which appears to have made
pcmcia services startup correctly at boot time but the cards are not
recognised and don't appear in the PCMCIA control panel.

Does anyone have any ideas or tips about getting these cards working? I have
a lonely Linux machine that can't talk to anyone...

Thanks,

Mark

It might help if you download the latest pcmcia cardservices source. Go
to http://pcmcia.sourceforge.org/ for info and the source. Then follow
the instruction to compile and install it. Restart pcmcia and see if it
helps. I don't know what version mandrake uses, but it's a revised
redhat, so it will probably be version 3.1.8. They are now currently at
3.1.17.

I had the same problem when I installed redhat 6.0 six months ago. It
was simply the fact that that version of pcmcia had no support for my
cards. Installed a new version and everything worked.
