Speicherprobleme mit memtest86 ermitteln

Supportdatenbank (thallma_memtest86)

Symptom

Ihr System läuft instabil und/oder es treten unvermittelte Systemabstürze auf.

Mögliche Ursache

Ein defekter Speicherbaustein kann dazu führen, daß Ihr System instabil wird bzw. Ihr System nach einiger Betriebszeit "einfriert", da auf einen defekten Speicherbereich zugegriffen wird.

Lösung

Sie können mit den folgenden Schritten einen Test des Arbeitsspeichers durchführen. Halten Sie dafür bitte eine leere Diskette bereit.

  1. Laden Sie sich von der folgenden Webseite das "Linux Memtest86 v2.4 Source Package" herunter:
    http://reality.sgi.com/cbrady_denver/memtest86/
  2. Entpacken Sie das Paket mit dem Befehl
    tar zxvf memtest86-x.y.tar.gz
  3. Wechseln Sie in das neu erstellte Verzeichnis
  4. Führen Sie den Befehl "make" aus.
  5. Melden Sie sich mit dem Befehl "su" als root an bzw. um.
  6. Legen Sie eine leere Diskette in Ihr Diskettenlaufwerk.
  7. Führen Sie den Befehl "make install" aus.
  8. Starten Sie den Rechner mit "shutdown -r now" neu und booten Sie von der Diskette.

Ohne weitere Angaben beim Start von memtest86 wird der Standard-Test durchgeführt. Um alle verfügbaren Tests durchführen zu lassen drücken Sie in der Hauptansicht von memtest die Taste c für configuration, wählen im erscheinenden Menü 3 für Test Selection und legen dann nochmals mit Taste 3 die Ausführung aller Tests (All Tests) fest. Der Nummernblock kann unter memtest nicht verwendet werden!

Hinweis: Dieses Programm gibt beim erfolgreichen testen des Speichers keine 100%ige Sicherheit das dieser wirklich ok ist!
