recognize the CDROM.
Is there people who could help me to understand what is wrong?

Thanks in advance for your help.

Here is what I have on my PC:

2 Hard drives(Samsung, PLS-31084A, 1.08 Gb each):
- both are on the IDE #1(connected on the mother board)
- IDE #2 is empty on the mother board

* Intel PIIX IDE controller(Bus mastering not supported)
  - IRQ = 14, 
    I/O ranges = 01F0-01F7 and 03F6-03F6 and 0170-0177 and 0376-0376
    with primary IDE controller(dual FIFO): 
    I/O ranges = 01F0-01F7 and 03F6 - 03F6
  - IRQ = 15, 
    I/O ranges = 3000-3007 and 3008-300F
    with secondary IDE controller(dual FIFO)
    I/O ranges = 0170-0177 and 0376-0376
  
1 CDROM(Creative Labs, 4x)
- IDE bus, connected on the Sound Blaster card(see below)
- IRQ = 10
- I/O = 0x168-016F and 036E-036F

1 Sound Blaster           
- IRQ = 5   Used by the audio interface
- A   = 220 Default value for the audio interface base I/O address
- D   = 1   Default value for the low DMA used by the audio interface
- H   = 5   Default value for the high DMA channel used used by the
audio interface
- P   = 330 Default value for MPU-401 interface's base I/O address
- T   = 6   Value to specify the card type

The documentation for the CDROM precises the following values:

primary I/O address for IDE port    = 1F0 (IRQ line = 14)
secondary I/O address for IDE port  = 170 (IRQ line = 15)
tertiary I/O address for IDE port   = 1E8 (IRQ line = 11 or 12)

Hi,
    When you installed the system, /dev/cdrom was created which is a
symbolic link to the real device.  To correct the problem, you will need to
know which device has your cdrom drive connected to it :-

    /dev/hda  - master on primary IDE interface
    /dev/hdb  - slave on primary IDE interface
    /dev/hdc  - master on secondary IDE interface
    /dev/hdd  - slave on secondary IDE interface

you may be able to ascertian the correct device from the messahes produced
by the kernel at boot time.
Alternatively, try the following command with each device in turn :-

    mount -t iso9660 <device> /mnt

when you find the correct device, indicated by a successful mount, do the
following command :-

    ln -sf <device> /dev/cdrom

    Martin
