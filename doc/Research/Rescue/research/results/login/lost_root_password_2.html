Simple yes but not the best way to go about it.  Try this way instead and 
you don't even need a boot disk (still a good idea to have one though.)  
If you use lilo, at the lilo image prompt (when you boot) type: "linux s" 
this assumes that linux is the name of the linux image.  This boots up in 
single user mode.  If you use loadlin, then add the word single to the 
end of the line in the BAT file that calls loadlin.  You may need to 
remount the / partition RW with this cmnd:
/sbin/mount -n -o remount,rw /
And then you can manually edit the /etc/passwd to remove the password.  
The root line should look something like this when you're done:
root::0:0:root:/root:/bin/bash
    ^^  Note the blank space.  
After doing this, you can run passwd to add a new one.  Type exit and you 
should be prompted for "Enter Runlevel" where you type the default 
runlevel that your system runs on.  Any questions?  Good.  Cheers.

He can also open the window and throw the box out!  If someone
has physical access to the hardware, the system isn't secure
from them.

Even easier (with LILO) is to boot in single user mode.  You
have to write the kernel image to the boot tracks of the hard
disk to prevent this from happening.  I'm not sure, but I 
don't think this is trivial.

but rather the root filesystem on the floppy.  And your "friend"
can make his own root/boot disk set before he sets foot in 
your office.  So in any case, paragraph 1 applies.
