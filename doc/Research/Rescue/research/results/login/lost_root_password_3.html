When you see lilo come up on the screen type `linux single`, assuming `linux`
is the label for your linux partition.  This will boot you into "single user"
mode.  There you ought to either be able to run `passwd` and change the
root password or edit your /etc/passwd file and delete the field which has
the encrypted password.   After you have deleted that field you can type
`init 3` and it will boot into linux normally and when you log in as root
there will be no password on the account.  The first thing you should do is
set a new password.  Hope this works/helps.

Saadiq

This won't necessarily work.  On Debian, for example, it asks you for
a root password before it will give you a root shell.  (I seem to
remember some commercial Unices doing the same.)  Fortunately, this is 
easy (more or less) to work around: boot with "linux init=/bin/sh".
Instead of trying to use /sbin/init, the first program the kernel will 
run is /bin/sh, rather conveniently with a UID of 0.  You'll need to
mount your root partition read/write, and if /usr is a separate
partition you might need to mount that as well.  Or you can edit
/etc/passwd by hand (mmm...ed... :-)  Then umount everything you can
and run /sbin/reboot.

Not even necessary.  

You boot from the floppy, mount the hd  (which you have to do anyway
in order to edit the passwd file) and set your PATH to include the dir
of the hd that holds vi.

e.g.: 
	<boot from floppy, and gain access to root prompt>
	# mount /dev/hd1a /mnt
	# PATH=$PATH:/mnt/bin:/mnt/usr/bin
	# vi /mnt/etc/passwd

The only other caveat is that if he has enabled shadow passwords, he
has to edit /etc/shadow instead.
