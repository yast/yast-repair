> By default, the lilo bootup is not password protected. So do
> linux 1

> prompt comes up at boot up.
> Thie will drop you into a root shell (without requireing a
> password) and you can then do passwd to enter a new password for
> root.
IMHO, single user mode should have a root password.  This is not the
case on most distributions, unfortunately.  To activate this feature,
add the following line to /etc/inittab.

su:1:wait:/sbin/sulogin

This should be placed after the other runlevel definitions, so that your
/etc/rc.d/* stuff is run first.

This, of course, makes it harder to recover from a lost root password. 
The answer is to NOT DO THAT.  :)

Seriously, you can boot from a rescue floopy, mount the root file
system, then edit /mnt/etc/shadow to remove the password.  Then reboot
from hard disk and set a new password.

I have found the slackware rescue disks to be the best.  They will work
even if you are using another distro and are available over the net.

Also, when ever you set root password, TEST IT *before* loging out as
root.

-Leolo
